# Containerized Monitoring Project

This is a college project demonstrating a containerized multi-service architecture using Docker and Docker Compose.

## Architecture
The project consists of two isolated microservices:
1. **Demo Application (`demo_app`)**: A Flask application that simulates a real-world web server. It contains various endpoints to generate different types of logs (`INFO`, `WARNING`, `ERROR`, `CRITICAL`), simulate traffic loads, and even mimic intentional application crashes.
2. **Monitoring API (`monitor_api`)**: An independent Flask service that parses and analyzes the logs generated by the `demo_app`. It provides endpoints to retrieve raw logs, filter them by severity levels, and generate a summary of the system's overall status.

Both applications run in their own Docker containers and communicate via a **Docker Shared Volume** mounted at `/app/logs`. This architecture allows the monitoring service to instantly access and evaluate the files generated by the demo app without relying on local system directories.

## Prerequisites
- Docker
- Docker Compose

## How to Run

1. Clone this repository.
2. Build and start the containers in the background:
   ```bash
   docker-compose up --build -d
   ```
3. The services will be accessible at:
   - **Demo App**: http://localhost:5000
   - **Monitor API**: http://localhost:8000

## Endpoints

### Demo App (Port 5000)
- `/success` : Generates an INFO log.
- `/error` : Generates an ERROR log.
- `/slow` : Simulates a slow request and generates a WARNING log.
- `/random` : Generates a random event (INFO, WARNING, or ERROR).
- `/load` : Simulates server load and tracks request count.
- `/health` : Checks system health.
- `/uptime` : Returns server uptime status.
- `/crash` : Intentionally crashes the app and generates a CRITICAL log.

### Monitor API (Port 8000)
- `/` : Root status endpoint.
- `/logs` : Returns the last 50 log entries in JSON.
- `/logs/<level>` : Filters the latest logs by severity level (e.g., `/logs/error`, `/logs/warning`).
- `/summary` : Returns a count summary of INFO, WARNING, and ERROR occurrences.

## Shutting Down
To safely stop the software and remove the containers, run:
```bash
docker-compose down
```
